# Umami Analytics

This folder should contain screenshots of the Umami web analytics dashboard.

## Required Screenshots

### 1. umami-analytics.png

Capture showing:

- Analytics dashboard overview
- Visitor statistics
- Page views
- Geographic distribution

## How to Capture

1. Navigate to `https://umami.yourdomain.com`
2. Select your website from the dropdown
3. Capture the dashboard

## Screenshots to Capture

### 1. Dashboard Overview

Main analytics view showing:

- Total visitors (today/week/month)
- Unique visitors
- Page views
- Bounce rate
- Average visit duration
- Today's realtime visitors

### 2. Visitors View

Detailed visitor analytics:

- Geographic distribution (world map)
- Country breakdown
- Device types (desktop/mobile/tablet)
- Browser statistics
- OS distribution

### 3. Behavior View

Page analytics:

- Most visited pages
- Entry/exit pages
- Referral sources
- Search keywords

### 4. Events View

Custom event tracking:

- Event categories
- Event actions
- Event counts

## Image Requirements

- **Format**: PNG or JPG
- **Resolution**: Minimum 1280x720
- **Theme**: Works well in both light and dark modes

## Naming Convention

- `umami-dashboard.png` - Main analytics overview
- `umami-visitors.png` - Visitor details
- `umami-behavior.png` - Page behavior
- `umami-events.png` - Event tracking

## Tips for Good Analytics Screenshots

1. Wait for some traffic data to accumulate
2. Show a mix of metrics
3. Include realtime visitor count if possible
4. Demonstrate geographic spread
5. Show multiple time ranges (today, 7 days, 30 days)

## Privacy Note

Umami is GDPR-compliant and doesn't require cookie consent. Screenshots can be shared publicly without privacy concerns since they don't contain personal data.
